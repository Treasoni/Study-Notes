# Skills（技能）

## 1. 什么是 Skills？

**Skills（技能）** 是一种预定义的可重用任务或工作流，它可以让 AI 快速执行特定的操作。

> 🎯 **生活比喻：快捷键**
>
> 想象你在用电脑：
> - 没有 Skills 时，每次保存文件都要点击「文件」→「保存」
> - 有了 Skills，就像按 `Ctrl+S` 一样，一键搞定
>
> Skills 就是 AI 世界的「快捷键」！

可以把 Skills 理解为「预编程的 AI 能力」——就像人类可以学习不同技能（开车、编程、做饭）一样，AI 也可以通过学习 Skills 来获得特定的执行能力。

## 2. Skills 与相关概念的区别

> 🎯 **餐厅比喻**
>
> - **Prompt** = 你对服务员说：「我要一份宫保鸡丁」
> - **Agent** = 会端菜、倒水、结账的全能服务员
> - **MCP Server** = 厨房里的厨师、洗碗工、采购员
> - **Skills** = 「点套餐」按钮，一键包含主食+饮料+甜点

[Prompt, Agent, MCP 是什么](Prompt,%20Agent,%20MCP%20是什么.md)

| 概念 | 定义 | 使用方式比喻 | 例子 |
|------|------|---------------|------|
| **Prompt** | 给 AI 的输入指令 | 每次都要告诉服务员要什么 | "帮我写一段 Python 代码" |
| **Agent** | 能调用工具的智能体 | 会联系厨房、后厨的智能服务员 | 能查询天气的 AI 助手 |
| **MCP Server** | 提供工具的服务端 | 厨房里的专业厨师团队 | 文件系统工具服务 |
| **Skills** | 预定义的完整任务 | 点套餐，一键搞定 | `/commit` 自动提交代码 |

## 3. Skills 的特点

> 🎯 **外卖套餐比喻**
>
> - 预定义 = 套餐搭配已经定好，不用自己选
> - 可组合 = 可以额外加饮料或小菜
> - 可配置 = 可以选择「微辣」或「不加香菜」

### 3.1 命令式调用
Skills 通常通过斜杠命令触发，就像命令行的快捷指令：
- `/commit` - 创建 git 提交
- `/review-pr` - 审查 Pull Request
- `/help` - 显示帮助信息

### 3.2 预定义行为
每个 Skill 都有预设的完整执行流程，AI 不需要每次重新思考如何完成任务。

> **比喻**：就像点外卖的「超值套餐」，里面的菜式、分量、价格都是固定的，不用每次都重新搭配。

### 3.3 可组合性
Skills 可以互相调用或与普通 Prompt 混合使用。

> **比喻**：套餐可以额外加饮料，或者「这个套餐再加一份小菜」。

### 3.4 可配置参数
Skills 可以接受参数来定制行为，例如 `/commit -m "修复登录bug"`。

> **比喻**：就像「宫保鸡丁套餐，微辣，不要花生」。还是那个套餐，但可以微调。

## 4. Skills 的工作原理

> 🎯 **ATM 机比喻**
>
> 你去银行取钱的流程：
> 1. 插卡并输入密码（触发 Skill）
> 2. ATM 识别你的账户（解析器识别命令）
> 3. 选择「取款 1000 元」功能（加载 Skill 定义）
> 4. ATM 连接到银行系统（发送给 AI）
> 5. 银行处理请求并扣款（执行工作流程）
> 6. ATM 吐出 1000 元现金（返回结果给用户）

```
┌─────────────────────────────────────────────────────────────────┐
│  用户触发 Skill（如 /commit）                                    │
│       ↓                                                          │
│  Skill 解析器识别命令                                            │
│       ↓                                                          │
│  加载对应的 Skill 定义（预配置的 prompt + 工作流程）             │
│       ↓                                                          │
│  将 Skill 定义发送给 AI                                          │
│       ↓                                                          │
│  AI 执行预设的工作流程（可能调用 Agent、Tools、MCP 等）          │
│       ↓                                                          │
│  返回执行结果给用户                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 6. Skills vs MCP

> 🎯 **智能音箱比喻**
>
> - **Skills** = 说「小爱同学，播放我的歌单」
> - **MCP** = 小爱同学背后的音乐服务 API
>
> 你不需要知道音乐服务怎么调用，只需要说命令。

| 维度 | Skills | MCP |
|------|--------|-----|
| **层级** | 用户交互层（看得见） | 工具服务层（看不见） |
| **触发方式** | 命令式（/command） | 由 Agent 调用 |
| **粒度** | 完整任务/工作流 | 单一工具功能 |
| **可发现性** | 列表可见，有文档 | 需要查询工具列表 |
| **比喻** | 「点套餐」按钮 | 后厨的厨师 |
| **典型用途** | 常用操作的快捷方式 | 提供可调用的函数 |

## 7. Skills 的优势

> 🎯 **手机 App 比喻**
>
> - **快速执行** = 点开微信直接聊天，不用每次输入网址
> - **一致性** = 每次打开微信界面都一样
> - **可发现** = App Store 可以搜索和查看所有 App
> - **低门槛** = 不用懂编程也能用微信
> - **可组合** = 微信可以分享文件到其他 App

1. **快速执行** - 无需每次重新描述任务，直接触发预定义流程
2. **一致性** - 相同操作总是以相同方式执行
3. **可发现** - 用户可以查看所有可用 Skills 及其文档
4. **低门槛** - 用户只需记住命令，无需了解内部实现
5. **可组合** - Skills 可以调用其他 Skills、Agent 或 MCP 工具

## 8. 如何创建自定义 Skill

> 🎯 **菜谱比喻**
>
> 创建自定义 Skill 就像写菜谱：
> - **name** = 菜名（宫保鸡丁）
> - **description** = 菜品介绍（川菜名菜，麻辣鲜香）
> - **parameters** = 原料（鸡肉、花生、辣椒...）
> - **prompt** = 烹饪步骤（先炒鸡丁，再加配菜...）
>
> 按照菜谱就能做出相同的菜！

### 8.1 基本结构

一个 Skill 通常包含以下部分：

```yaml
# skill.yaml
name: my-skill
description: 这是我自定义的技能
parameters:
  - name: input
    type: string
    required: true
prompt: |
  请帮我处理以下内容：{{input}}
  要求：
  1. 格式化输出
  2. 检查语法错误
```

### 8.2 实现方式（以 Claude Code 为例）

Claude Code 支持 User Skills，用户可以定义自己的技能：

**步骤：**
1. 在配置目录创建 `.claude/skills/` 文件夹
2. 创建 `.yaml` 或 `.py` 文件定义 Skill
3. 使用时通过 `/skill-name` 触发

**示例（Python Skill）：**

```python
# .claude/skills/analyze.py

def execute(context, args):
    """
    分析当前代码库结构

    Args:
        context: 运行上下文（包含文件系统、代码信息等）
        args: 用户传入的参数
    """
    files = context.get_files("**/*.py")
    analysis = []

    for file in files:
        analysis.append({
            "path": file.path,
            "lines": len(file.lines),
            "functions": extract_functions(file.content)
        })

    return format_analysis(analysis)
```

## 9. 完整示例对比

> 🎯 **做饭比喻**
>
> **传统方式**：每次做饭都要从零开始——洗菜、切菜、配料、炒菜...
> **Skill 方式**：直接点外卖，5 分钟到货！

### 传统方式 vs Skill 方式

**任务：创建 git 提交**

**传统方式：**
```
用户: 帮我创建一个 git 提交，包括我修改的所有文件，
     分析这些修改内容，生成一个合适的提交消息，
     不要提交 .env 文件
AI: （每次都要重新理解这些要求）
```

**Skill 方式：**
```
用户: /commit
AI: （自动执行预定义的完整提交流程）
```

> 效果对比：
> - 传统方式：每次输入 50+ 字符
> - Skill 方式：只需输入 `/commit`（8 个字符）
> - 效率提升：**6 倍以上**！

## 10. 总结

Skills 是连接用户和 AI 能力的桥梁：

> 🎯 **盖房子比喻**
>
> ```
> 用户（业主）
>     ↓ 说：「我要盖一栋房子」
> ┌─────────────────────────────────────────────┐
> │  Skills Layer (装修套餐)                   │
> │  - 「精装套餐」、「豪装套餐」               │
> └─────────────────────────────────────────────┘
>     ↓ 选择套餐
> ┌─────────────────────────────────────────────┐
> │  Agent Layer (项目经理)                     │
> │  协调各个工种，按计划施工                  │
> └─────────────────────────────────────────────┘
>     ↓ 分配任务
> ┌─────────────────────────────────────────────┐
> │  MCP Layer (专业工种)                       │
> │  - 水电工、泥瓦工、油漆工、木工            │
> └─────────────────────────────────────────────┘
>     ↓ 执行工作
> ┌─────────────────────────────────────────────────────┐
> │  AI Model (核心大脑)                           │
> │  理解需求，设计方案，统筹协调                 │
> └─────────────────────────────────────────────────────┘
> ```

```
用户需求
    ↓
┌─────────────────────────────────────────────┐
│  Skills Layer (预定义的任务工作流)           │
│  - /commit, /test, /build 等                │
└─────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────┐
│  Agent Layer (协调 AI 和工具)                │
└─────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────┐
│  MCP Layer (提供标准化工具服务)              │
│  - Filesystem, Git, GitHub, Weather 等      │
└─────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────┐
│  AI Model (大语言模型)                       │
└─────────────────────────────────────────────┘
```

**核心价值：**
- **对用户**：简化常用操作，提高效率
- **对开发者**：封装最佳实践，保证一致性
- **对 AI**：减少理解成本，专注执行
