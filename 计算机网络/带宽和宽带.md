**一句话先记住**

>- **宽带**：你办的“网络套餐 / 上网通道”
>- **带宽**：这条通道**有多宽、一次能跑多少数据**

# 1. 带宽和宽带
## 1.1  宽带是什么？

👉 **宽带 = 一条高速公路**

>- 你家装的 **100M / 300M / 1000M 宽带**
>- 移动 / 电信 / 联通给你拉的一根网线
>- 本质：**能上网这件事本身**

> 没有宽带，就等于 **没路**，车根本跑不了。


## 1.2 带宽是什么？

👉 **带宽 = 高速公路有几条车道**

>- 带宽越大 = 车道越多
>- 同一时间能跑的数据就越多
>- 下载更快、看视频更流畅、多设备不卡

> 带宽小 = 单车道  
> 带宽大 = 八车道高速


## 1.3 两则关系区分
## 1.3.1 宽带（我们日常说的）

> **宽带 = 套餐标称的下行速率（下载能力）**

比如：

>- 100M 宽带
>- 300M 宽带
>- 1000M 宽带

👉 **默认说的都是下载**

### 1.3.2  带宽（更完整的概念）

> **带宽 = 上行 + 下行 的传输能力**

也就是：

>- 下行带宽（下载）
>- 上行带宽（上传）

只是现实中：

>- **下行被拿出来当“卖点”**
>- **上行被弱化甚至隐藏**

# 2. 很多人容易混淆的一点（重点‼️）

## 2.1 你办的「100M宽带」

👉 **M = Mbps（兆比特）**

但你电脑里显示的下载速度是：  
👉 **MB/s（兆字节）**

## 2.2 换算关系

`1 MB/s ≈ 8 Mbps`

所以：

>- 100M 宽带
>- 理论下载速度 ≈ **12.5 MB/s**

> 看到只有 10~12 MB/s 是**正常的**，不是被坑了

### 2.2.1 换算关系讲解

**100M 宽带 ≈ 12.5 MB/s？**
 **关键点只有一句话（先背）**

> **1 Byte（字节） = 8 bit（比特）**
####  1️⃣ 宽带运营商说的 **M** 是什么？

>- **100M 宽带**
>- 这个 **M = Mbps**
>- 全称：**Megabit per second（兆比特 / 秒）**

👉 单位是 **bit（比特）**

#### 2️⃣ 电脑显示的下载速度是啥？

>- 浏览器 / 迅雷 / Steam
>- 显示的是 **MB/s**
>- 全称：**Megabyte per second（兆字节 / 秒）**

👉 单位是 **Byte（字节）**

#### 3️⃣ Bit 和 Byte 的关系（核心）

`1 Byte (B) = 8 bit (b)`

#### 4️⃣ 正式计算过程（一步不跳）

`100 Mbps ÷ 8 = 12.5 MB/s`

所以：

> **100M 宽带 → 理论最大下载 ≈ 12.5 MB/s**
#### 5️⃣ 为什么说是「理论」？

因为现实中还会被这些吃掉一部分：

>- 网络协议开销（TCP / IP）
>- 路由器性能
>- WiFi 干扰
>- 服务器限速

👉 实际 **10 ~ 11 MB/s** 都是正常的
### 2.2.2 再给你一个“秒懂记忆法” 🧠

**心法口诀**

> **看到 b 就除 8，看到 B 就乘 8**

>- Mbps → MB/s：**÷ 8**
>- MB/s → Mbps：**× 8**

# 3. 那什么时候「只看宽带」就不够了？

只要你做了下面任何一件事👇  
**就必须开始看带宽（尤其是上传）**，看看[网络宽带的上传和下载](网络宽带的上传和下载.md)

## 3.1 有 NAS / 私有云

>- 外网访问文件
>- 照片、视频同步
>- Web 服务

👉 你用的是：

> **家里宽带的“上传带宽”**

## 3.2 自建服务 / Docker

>- Home Assistant
>- 自建博客
>- API 服务
>- 内网穿透

👉 本质：

> 你在“给别人提供数据”

## 3.3 直播 / 推流 / 视频会议

>- OBS
>- 腾讯会议
>- 远程桌面
>- 远程开发

👉 对 **上传稳定性** 极度敏感

## 3.4 多设备同时用

>- 家人看视频
>- 你在上传 / 备份
>- NAS 同步云端

👉 上行一满，全家都卡

# 4. 一个特别重要、但容易被忽略的点 ⚠️

## 4.1  ❌ “宽带越大 = 网一定越快”

**这是错的**

因为：

>- 宽带 ≈ 套餐名
>- 带宽 ≈ 实际能力

举个扎心例子 👇

|套餐|下载|上传|
|---|---|---|
|1000M 家宽|1000M|30M|
|300M 商宽|300M|300M|

👉 对 NAS / 服务：

- **300M 商宽完胜**
- 千兆家宽反而被吊打
## 4.2  正确的“看法”应该是这样（方法论）

 **先问自己一句话**

> **我只是“上网”，还是“对外提供服务”？**> 

**对应选择重点**
- **✔ 只是用：**
	- 👉 **看宽带（下载）**

- ✔ 有输出：
	👉 **看带宽结构**
	- 上行多少？
	- 是否对称？
	- 稳不稳定？
# 5. 日常最常见的网络使用场景

## 5.1  刷网页 / 聊天（最低需求）

|场景|所需速率|
|---|---|
|微信文字 / 图片|< 1 Mbps|
|普通网页浏览|1–2 Mbps|
|在线文档|1–3 Mbps|

👉 几乎感觉不到“网速”的存在

## 5.2 看视频（大家最关心的）

|视频清晰度|需要带宽|
|---|---|
|480P|1–2 Mbps|
|720P|3–5 Mbps|
|1080P|6–10 Mbps|
|2K|15–20 Mbps|
|4K|25–50 Mbps|

👉 一个 **100M 宽带**  
同时看 **3–5 个 1080P** 完全没问题

## 5.3  音乐 / 语音

|场景|带宽|
|---|---|
|在线音乐|< 1 Mbps|
|微信语音|< 0.5 Mbps|
|高清语音通话|1–2 Mbps|


## 5.4  游戏（你可能会误解的一点）

|游戏行为|带宽|关键点|
|---|---|---|
|联机对战|< 1 Mbps|**看延迟，不看带宽**|
|游戏更新 / 下载|20–100 Mbps|吃下载|

👉 打游戏卡 ≠ 网速慢  
👉 多半是 **延迟 / 丢包**

### 5.5 视频会议 / 网课

|场景|下载|上传|
|---|---|---|
|普通会议|2–3 Mbps|1–2 Mbps|
|高清视频会议|4–6 Mbps|3–4 Mbps|
|多人视频|6–10 Mbps|5–8 Mbps|

⚠️ 这里开始**吃上传了**
## 5.6  一个非常实用的“判断口诀” 🧠

> **看视频 → 看下载  
> 打游戏 → 看延迟  
> 会议 / NAS → 看上传  
> 局域网 → 看网口**  [网口的作用](网口的作用.md)

**🔹 普通个人 / 学生**

- 100M–300M
- 完全够用

**🔹 多人家庭**

- 300M–500M
- 更稳

**🔹 折腾党 / NAS / 推流**

- 看 **上传**
- 50M 起步，越多越好
