#网络 

先看[带宽和宽带](带宽和宽带.md)对这一方面有了解。
**先给结论（你先有个“正确认知”）**

> **家用宽带 = 重度偏向“下载”，严重限制“上传”**

所以你会遇到：

>- 上传文件慢 ❌
>- NAS 外网访问慢 ❌
>- 自建服务、推流、远程桌面卡 ❌

不是你设备差，**是宽带设计就是这样**
# 1. 为什么下载快、上传慢？（核心原因）

## 1.1 宽带本来是给「看内容的人」用的

运营商假设你主要在：

>- 刷视频
>- 看直播
>- 下载游戏
>- 网页浏览

这些行为的共同点是：

> **你在“收数据”，不是“发数据”**

所以：

- 下载给得多
- 上传给得少  
    （省成本，还符合大多数用户习惯）
    

## 1.2 技术层面的原因（不讲太深）

家用宽带大多是：

>- **PON / 光纤共享**
>- 一根光纤 → 分给一片小区

👉 上传如果不限制，会发生：

>- 大量用户同时“往外发”
>- 上行链路直接被打满
>- 整个小区都卡

所以运营商干脆：

> **强行限制每户的上传带宽**

# 2. 为什么局域网很快，外网却很慢？

这个反差很多人懵过 😄

## 2.1 局域网（家里）

>- 千兆网口
>- 实际：100–110 MB/s

### 外网（互联网）

>- 被宽带限制
>- 上传：2.5 MB/s

👉 差 **40 倍以上**

不是你配置错  
是**物理层级不一样**
# 3. 局域网内的影响
**一句话结论（先记住）**
> **在局域网里，根本不存在“下载 / 上传谁更重要”的说法  
> 只有一句话：你在“收数据”，还是在“发数据”。**

也就是说：  
**局域网是对称的，不像宽带那样偏向下载**
## 3.1 局域网里的“上传 / 下载”到底是怎么回事？

 **举一个最直观的例子 👇**
**场景：电脑从 NAS 拷文件**

```
NAS  →  电脑
```

>- 对电脑：下载
>- 对 NAS：上传
>- 对网络：**只是单向传输**

**反过来：电脑往 NAS 拷**

```
`电脑  →  NAS`
```

>- 对电脑：上传
>- 对 NAS：下载
>- 对网络：**能力完全一样**
👉 速度几乎不会变

# 4. 为什么局域网“上传不慢”？

因为
## 1️⃣ 网口是对称的

- 千兆 = 上下各 1Gbps
- 2.5G = 上下各 2.5Gbps

不像宽带那样被掐上行

## 2️⃣ 没有运营商限速

- 不共享小区出口
- 不限 QoS
- 不区分“你发”还是“你收”

## 3️⃣ 延迟极低

- 局域网延迟：0.1–1 ms
- 几乎不影响吞吐
# 5. 那局域网有没有“瓶颈”？有，但不是方向问题

 **常见瓶颈列表（很重要）**

|瓶颈|影响|
|---|---|
|百兆网口|上下都慢|
|网线太差|丢包 / 降速|
|交换机太弱|并发不行|
|硬盘慢|网络跑不满|
|CPU 弱|加密 / 校验卡|

👉 **跟上传 / 下载无关**
# 6. PVE 的实时上传 / 下载速率？
# 6.1 `iftop`（最直观，强烈推荐 ⭐）

**安装**

```bash
apt update apt install -y iftop
```

**查看某个网口（比如 vmbr0）**

```bash
iftop -i vmbr0
```

你会看到：

>- 左边：连接 IP
>- 右边：实时速率（上传 / 下载）
>- 单位：Kb / Mb / Gb

👉 **谁在吃你上行，一眼就能看到**
