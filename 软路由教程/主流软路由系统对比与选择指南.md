# 主流软路由系统对比与选择指南

> [!info] 概述
> **一句话定义**：软路由系统是运行在通用硬件（如x86迷你主机、ARM盒子）上的路由器操作系统，替代传统硬路由器实现网络转发、防火墙、NAT等功能。
>
> **🎯 通俗比喻**：如果硬路由是"功能机"（出厂就固定功能），软路由就是"智能手机"（可以自由安装APP、扩展功能）。

---

## 核心概念

### 是什么

软路由系统是专门设计用于网络设备的操作系统，可运行在通用x86/ARM硬件上，提供路由、防火墙、VPN、流量控制等网络功能。

### 为什么需要

- **功能强大**：支持企业级防火墙、多WAN负载均衡、科学上网、Docker容器等
- **灵活扩展**：可安装各种插件和软件包
- **硬件自主**：可自由选择性能更强的硬件
- **成本效益**：相比企业级设备节省大量成本

### 通俗理解

**🎯 比喻**：
- **硬路由** = 品牌台式机（联想、戴尔），配置固定，功能有限
- **软路由** = 自己组装的电脑，想怎么配就怎么配，想装什么系统都行

**📦 硬件示例**：
```
最低配置：J4125 + 4GB内存 + 双千兆网口（约200-300元）
推荐配置：N5105/N6000 + 8GB内存 + 2.5G网口（约500-800元）
进阶配置：i3/i5 + 16GB内存 + 多网口（约1000元+）
```

#### OpenWrt 系详细对比

**iStoreOS 特点**：
- 基于OpenWrt官方Release分支，100%兼容
- 中文界面，对新手友好
- 内置NAS功能（文件共享、媒体服务器）
- iStore应用商店，一键安装插件
- 更适合家庭和小型企业使用

**ImmortalWRT 特点**：
- 移植更多软件包，支持更多设备
- 针对中国大陆用户优化
- 集成iStore应用商店
- 内置应用过滤、在线用户管理

**高大全版（eSir编译）**：
- 基于6.1内核
- 插件极其丰富，被称为"高大全"
- 适合作为旁路由使用
- 固件体积较大

## 软路由部署模式对比

### 主路由模式

**拓扑结构**：
```
光猫 → 软路由 → AP/交换机 → 终端设备
```

**优点**：
- ✅ 网络架构简单，单网关出口
- ✅ 对内网设备完全透明，无需额外配置
- ✅ 性能释放完整，充分利用硬件
- ✅ 功能完整，所有路由特性可用

**缺点**：
- ❌ 替换现有主路由，改动较大
- ❌ 稳定性要求极高，故障影响全网
- ❌ 无线功能需额外AP配合
- ❌ 高带宽场景可能存在瓶颈

**适用场景**：新建网络、追求最佳性能、网络玩家

---

### 旁路由模式

**拓扑结构**：
```
光猫 → 主路由 → 软路由（旁路）→ 终端设备（指定网关）
```

**优点**：
- ✅ 保留原主路由，改动最小
- ✅ 单网口设备也能使用（如N1盒子）
- ✅ 故障影响范围可控，可自动回退
- ✅ 灵活分流，按需处理流量

**缺点**：
- ❌ 配置相对复杂，容易设置不当
- ❌ 数据来回转发，性能有损耗
- ❌ 部分功能受限（如DHCP、端口映射）
- ❌ 高性能软路由做旁路由有点"大材小用"

**适用场景**：已有稳定主路由、想尝试软路由、小白入门

---

### 虚拟机模式（PVE/ESXi）

**拓扑结构**：
```
物理机（PVE）→ 虚拟机（软路由）→ 网络
```

**优点**：
- ✅ 硬件利用率高，一机多用
- ✅ 隔离性好，故障互不影响
- ✅ 快照备份，折腾无忧
- ✅ 灵活扩展，可随时增减功能

**缺点**：
- ❌ 需要一定虚拟化知识
- ❌ PCI直通配置可能复杂
- ❌ 性能有虚拟化损耗（约5-10%）
- ❌ 网络配置相对复杂

**适用场景**：多功能需求、学习折腾、资源有限

---

### All-in-One 模式

**拓扑结构**：
```
一台机器 → NAS + 软路由 + 服务器 + Docker...
```

**优点**：
- ✅ 极致硬件整合，功耗低、体积小
- ✅ 成本效益最高
- ✅ 统一管理，便于维护

**缺点**：
- ❌ **主路由不建议放AIO**（网络是核心，必须稳定）
- ❌ 单点故障风险高
- ❌ 性能分配需要权衡

**推荐方案**：
- 旁路由放AIO（虚拟机）
- 物理主路由单独部署
- 做故障自动转移机制

## 如何选择软路由系统

### 决策树

```
开始
  │
  ├─ 你的主要用途是什么？
  │   ├─ 家庭网络 + 科学上网 → OpenWrt系
  │   ├─ 企业防火墙 + 安全防护 → OPNsense/pfSense
  │   ├─ 专业网络工程 → RouterOS
  │   └─ 学习折腾 → OpenWrt/OPNsense
  │
  ├─ 你的技术水平？
  │   ├─ 小白 → iStoreOS
  │   ├─ 进阶 → ImmortalWRT
  │   └─ 专家 → OpenWrt官方/OPNsense
  │
  ├─ 硬件配置？
  │   ├─ 低端（2GB内存）→ OpenWrt系
  │   ├─ 中端（4-8GB）→ OpenWrt/OPNsense
  │   └─ 高端（16GB+）→ 全系列可选
  │
  └─ 功能需求？
      ├─ 需要WiFi → OpenWrt系
      ├─ 需要防火墙 → OPNsense/pfSense
      ├─ 需要NAS → iStoreOS
      └─ 需要科学上网 → OpenWrt系
```

### 选择清单（决策因素）

#### 1. 明确需求

**✅ 需求分析检查清单**：
- [ ] 主要用途是什么？（科学上网/防火墙/NAS/学习）
- [ ] 网络带宽多少？（千兆/2.5G/万兆）
- [ ] 有多少设备？（10/50/100+）
- [ ] 需要WiFi吗？（是→需要AP）
- [ ] 需要哪些功能？（VPN/AdGuard/Docker）

#### 2. 硬件评估

**✅ 硬件要求参考**：
```
OpenWrt系：    J4125 + 2GB + 双千兆网口
OPNsense：     N5105 + 4GB + 双千兆网口

```

#### 3. 技术水平

**✅ 技能匹配**：
- 小白 → iStoreOS
- 有经验 → ImmortalWRT
- 技术背景 → OPNsense/OpenWrt官方

#### 4. 部署模式

**✅ 模式选择**：
- 新建网络 → 主路由模式
- 已有设备 → 旁路由模式
- 多功能需求 → 虚拟机模式

## 避坑指南

### ⚠️ 常见错误

1. **盲目追求高性能**
   - 100-300元价位可满足4人家庭需求
   - 不要为难钱包，按需选择

2. **选择已停止维护的系统**
   - ❌ Koolshare LEDE 已停止更新
   - ✅ 选择 iStoreOS 或 ImmortalWRT

3. **主路由放All-in-One**
   - 网络是核心，必须稳定
   - 主路由建议物理独立

4. **忽视安全性**
   - 及时更新系统
   - 配置防火墙规则
   - 修改默认密码

### ⚠️ 硬件避坑

1. **网口数量**
   - 至少需要2个网口
   - 主路由建议多网口

2. **WiFi期望**
   - 软路由WiFi功能较弱
   - 建议单独购买AP

3. **散热问题**
   - 袖珍刀版要注意散热
   - 建议带风扇版本

## 最佳实践

### 2025-2026年趋势建议

1. **高带宽用户（≥2000M）**
   - 考虑"物理直装"
   - 放弃虚拟化和旁路由
   - 追求极致性能

2. **普通家用**
   - 硬路由做主路由 + 软路由旁路由（或虚拟机旁路由）
   - 最佳平衡方案

3. **All-in-One玩家**
   - 旁路由放AIO
   - **主路由物理独立**
   - 网络稳定优先

4. **新手建议**
   - 从旁路由开始
   - 熟悉后再考虑更复杂的架构

### 核心原则

> **"网络是核心中的核心，一定要可靠不折腾"**

- 主路由要稳定：物理设备 > 虚拟机
- 旁路由要灵活：虚拟机 > 物理设备
- All-in-One定位明确：适合旁路由、NAS、服务器，**不适合主路由**
