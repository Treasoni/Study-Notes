# 旁路由原理详解

> [!info] 概述
> **一句话定义**：旁路由是挂在主路由旁边的"代理网关"，通过修改设备的网关设置，让流量先经过旁路由处理后再转发到主路由。
> **通俗比喻**：如果把主路由比作小区大门，旁路由就是小区门口的"安检站"——你要出门必须先过安检，安检通过后再由大门放行。

## 核心概念

### 什么是旁路由？

旁路由（学名：**旁路网关**）是一种网络配置方式，它在主路由器旁边接入一台额外的设备（软路由），专门负责特定的网络任务，如代理翻墙、广告过滤、流量控制等。

**关键特点**：
- 旁路由**不直接连接互联网**，而是通过主路由器访问网络
- 旁路由通过 **LAN 口**与主路由连接（不是 WAN 口）
- 旁路由与主路由在**同一个网段**

### 为什么需要旁路由？

| 场景 | 说明 |
|------|------|
| 主路由不能动 | 运营商光猫一体机、租房不能改网络 |
| 主路由性能弱 | 主路由跑不动插件（如科学上网） |
| 部分设备需要代理 | 只有某些设备需要翻墙，其他设备直连 |
| 灵活切换 | 旁路由挂了不影响基本网络，拔掉就恢复 |

> [!tip] 核心观点
> **只有你不想动主路由，或者主路由不支持改动时，才考虑使用旁路由**。否则直接用主路由更简单。

## 旁路由工作原理（核心！）

### 网络拓扑图

```
                    ┌──────────────┐
                    │   互联网     │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │   主路由     │  ← IP: 192.168.1.1
                    │  (光猫/硬路由)│
                    └──────┬───────┘
                           │ LAN口
              ┌────────────┼────────────┐
              │            │            │
        ┌─────▼─────┐ ┌────▼────┐ ┌─────▼─────┐
        │  旁路由   │ │ 设备 A  │ │  设备 B   │
        │ (软路由)  │ │(网关指向│ │ (网关指向 │
        │192.168.1.2│ │ 旁路由) │ │  主路由)  │
        └───────────┘ └─────────┘ └───────────┘
```

### 网关设置的本质原理

> [!important] 🔑 核心原理
> **网关 = 设备发送数据包的"第一站"**

当你的设备（手机/电脑）要上网时：
1. 设备根据**路由表**决定把数据包发给谁
2. **默认网关**就是"不知道发谁时，就发给它"
3. 网关收到数据后，再根据**它的路由表**决定下一步

**这就是为什么设置网关=旁路由就能让流量经过旁路由！**

### 数据流向对比

#### 设备网关指向主路由（不走旁路由）
```
手机 ──数据包──▶ 主路由 ──▶ 互联网
                  ↑
            直接转发，不经过旁路由
```

#### 设备网关指向旁路由（走旁路由）
```
手机 ──数据包──▶ 旁路由（处理：代理/过滤）──▶ 主路由 ──▶ 互联网
                  ↑                      ↑
            第一站：先到旁路由      旁路由再转发给主路由
```

### 通俗比喻：快递中转站

```
📦 你的包裹（数据包）

方案A：直邮
  你 ──▶ 快递总站（主路由）──▶ 目的地

方案B：经过安检站（旁路由）
  你 ──▶ 安检站（旁路由）──▶ 快递总站（主路由）──▶ 目的地
         ↑
    在这里拆包检查、换包装（代理/过滤）
```

## 配置要点

### 旁路由基本配置

| 配置项 | 设置值 | 说明 |
|--------|--------|------|
| 旁路由 IP | 192.168.1.2 | 与主路由同网段，不冲突 |
| 旁路由网关 | 192.168.1.1 | **指向主路由**（关键！） |
| 旁路由 DNS | 192.168.1.1 | 指向主路由或公共 DNS |
| 旁路由 DHCP | **关闭** | 避免与主路由冲突 |

> [!warning] ⚠️ 关键点
> **旁路由的网关必须指向主路由！** 这样旁路由处理完数据后才知道往哪里发。

### 终端设备网关设置

#### 方式一：手动指定（推荐新手）

在需要走旁路由的设备上，手动修改网络设置：

```
IP 地址：    192.168.1.100（同网段任意未占用 IP）
子网掩码：   255.255.255.0
网关：       192.168.1.2  ← 填旁路由的 IP！
DNS：        192.168.1.2  ← 也填旁路由的 IP
```

**优点**：
- 不影响其他设备
- 想走旁路由就改，不想走就不改
- 旁路由挂了不影响其他设备

**缺点**：
- 每个设备都要手动设置

#### 方式二：DHCP 下发（全局模式）

在**主路由**的 DHCP 设置中：

```
网关： 192.168.1.2  ← 改成旁路由 IP
DNS：  192.168.1.2  ← 改成旁路由 IP
```

**优点**：
- 所有设备自动走旁路由
- 无需逐个设置

**缺点**：
- 旁路由挂了，所有设备都上不了网

### 为什么旁路由要关闭 DHCP？

```
问题场景：
┌─────────┐    ┌─────────┐
│ 主路由   │    │ 旁路由   │
│ DHCP 开  │    │ DHCP 开  │  ← 两个 DHCP 同时工作！
└────┬────┘    └────┬────┘
     │              │
     └──────┬───────┘
            │
       新设备接入
            ↓
    "我该听谁的？" 😵
```

**后果**：
- 设备可能拿到错误的网关
- IP 地址冲突
- 网络不稳定

**解决方案**：只保留一个 DHCP（通常是主路由）

## 常见问题排查

### 问题一：设置了网关但上不了网

**可能原因**：
1. 旁路由的网关没指向主路由
2. 旁路由防火墙阻止了转发
3. IP 地址冲突

**解决方法**：
```bash
# 在旁路由（OpenWrt）上检查网关设置
cat /etc/config/network | grep gateway

# 确保防火墙允许转发
# 或者添加 SNAT 规则
iptables -t nat -I POSTROUTING -o br-lan -j MASQUERADE
```

### 问题二：部分网站打不开

**可能原因**：
- 旁路由的代理规则问题
- DNS 解析问题

**解决方法**：
1. 检查旁路由的代理插件配置
2. 尝试将 DNS 也设置为旁路由 IP

### 问题三：旁路由挂了全部断网

**原因**：使用了 DHCP 全局模式，所有设备的网关都是旁路由

**解决方法**：
1. 临时：拔掉旁路由网线，重启主路由
2. 长期：改用手动指定模式，只让需要的设备走旁路由

### MASQUERADE 是什么？为什么要加？

```bash
iptables -t nat -I POSTROUTING -o br-lan -j MASQUERADE
```

**作用**：把数据包的**源 IP 改成旁路由的 IP**

**为什么需要**：
```
没有 MASQUERADE：
手机(192.168.1.100) ──▶ 旁路由 ──▶ 主路由 ──▶ 互联网
                              │
                         数据包源 IP 还是 192.168.1.100
                         主路由看到："这不是我网的设备！"
                         可能会丢弃（IP/MAC 校验）

有 MASQUERADE：
手机(192.168.1.100) ──▶ 旁路由 ──▶ 主路由 ──▶ 互联网
                              │
                         数据包源 IP 变成 192.168.1.2（旁路由）
                         主路由看到："这是旁路由发来的，放行！"
```

## 架构对比

### 旁路由 vs 主路由

| 对比项 | 主路由模式 | 旁路由模式 |
|--------|-----------|-----------|
| 网络位置 | 网络出口，所有流量必经 | 并行于主路由，可选经过 |
| 改动程度 | 需要替换现有路由器 | 不改动现有网络 |
| 故障影响 | 全网断网 | 仅影响走旁路由的设备 |
| 性能 | 最佳 | 多一跳，略有延迟 |
| 灵活性 | 低 | 高（可随时切换） |

### 旁路由的优缺点

| 优点 | 缺点 |
|------|------|
| 不改变原有网络拓扑 | 多一跳，性能略有损失 |
| 旁路由故障不影响基本网络 | 需要额外硬件 |
| 可以只让部分设备走旁路由 | 配置相对复杂 |
| 方便测试和实验 | 可能遇到非对称路由问题 |

## 最佳实践

### 推荐配置方案

```
✅ 推荐：手动指定 + 旁路由关闭 DHCP

主路由：
  - IP: 192.168.1.1
  - DHCP: 开启
  - 网关/DNS: 默认（自身）

旁路由：
  - IP: 192.168.1.2
  - 网关: 192.168.1.1 ← 指向主路由
  - DNS: 192.168.1.1
  - DHCP: 关闭

需要代理的设备：
  - 网关: 192.168.1.2 ← 手动指定旁路由
  - DNS: 192.168.1.2

不需要代理的设备：
  - 自动获取（走主路由）
```

### 配置检查清单

- [ ] 旁路由 IP 与主路由同网段且不冲突
- [ ] 旁路由网关指向主路由 IP
- [ ] 旁路由 DHCP 已关闭
- [ ] 旁路由防火墙允许转发（或添加 MASQUERADE）
- [ ] 终端设备网关/DNS 指向旁路由 IP
