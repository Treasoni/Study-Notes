# CLAUDE.md - AI 学习知识库

## 项目概述
这是一个用于学习 AI 相关知识的 Obsidian 知识库。

## 学习原则

### 1. 优先使用官方资源
当学习新知识点时，必须按以下顺序获取信息：
1. **官方文档** - 使用 WebSearch 搜索 "xxx official documentation"
2. **官方 GitHub** - 查找最新的 API 变更和示例
3. **权威博客** - 官方博客或核心维护者的文章
4. **社区资源** - 仅作为补充

### 2. 通俗易懂的解释
无论用户要求初级、中级还是高级内容，都必须包含：
- **一句话定义**：简洁的核心概念
- **通俗比喻**：用日常生活中的例子类比
- **具体示例**：可运行的代码或操作步骤
- **图示说明**：使用 ASCII 图或 Mermaid 图表

### 3. 知识关联
- 每个新知识点都要检查与现有笔记的关联
- 自动添加双向 wikilink
- 更新相关笔记的"相关文档"部分

## 自定义命令

### /learn <主题>
学习一个新知识点的工作流程：

1. **搜索官方资源**
   - 使用 WebSearch 搜索 "{主题} official documentation 2026"
   - 使用 WebFetch 获取官方文档核心内容

2. **创建笔记**
   - 在合适的目录下创建笔记文件
   - 使用标准笔记模板（见下方）

3. **建立关联**
   - 搜索知识库中相关内容
   - 添加双向 wikilink
   - 更新 MOC 索引

### /update <文件路径>
更新现有笔记的工作流程：

1. **读取现有内容**
   - 读取指定文件
   - 分析当前结构和内容
   - **识别并标记用户自定义章节**（见下方保护列表）

2. **获取最新信息**
   - 识别笔记主题
   - 使用 WebSearch 搜索官方最新文档
   - 使用 WebFetch 获取更新内容

3. **增量更新（重要）**
   - **仅更新**：技术参数、API 变更、最佳实践、参考资料
   - **禁止修改**：用户自定义章节（见保护列表）
   - 保持原有格式和风格

4. **检查双向链接**
   - 扫描知识库中引用此文件的其他笔记
   - 确保链接正确
   - 添加新的关联链接

#### 🛡️ 用户内容保护列表
以下章节在更新时**必须完整保留**，不得修改或删除：
- `## 个人笔记` / `## My Insights`
- `## 随手记` / `## Quick Notes`
- `## 学习心得` / `## Learnings`
- `## 踩坑记录` / `## Pitfalls`
- `## 待探索` / `## TODO`
- 任何以 `> [!personal]` 开头的 callout 块

### /organize <文件夹路径>
整理文件夹的工作流程：

1. **调用 knowledge-base-organizer skill**
   - 扫描指定文件夹下的所有文件
   - 分析文件类型和内容

2. **孤岛检测（Island Detection）**
   - 扫描整个知识库，识别没有任何入链（Inbound links）的笔记
   - 生成"孤岛笔记"列表
   - 为每个孤岛笔记：
     - 查找语义相关的其他笔记
     - 在相关笔记或 MOC 中添加引用
     - 确保每个笔记至少有一个入链

3. **生成/更新 MOC**
   - 创建或更新文件夹的 MOC 索引
   - 按主题分类整理
   - 添加快速查找表
   - **将孤岛笔记优先展示在 MOC 中**

4. **建立文档关联**
   - 识别相关文档
   - 添加双向 wikilink
   - 更新标签分类

## 注意事项

1. **年份敏感**：搜索时始终使用当前年份（2026）
2. **链接规范**：使用 `[[文件夹/文件名]]` 格式，不带 .md 后缀
3. **标签规范**：使用小写英文，用连字符分隔
4. **保持简洁**：CLAUDE.md 控制在 150-200 条指令以内
5. **保护用户内容**：任何更新操作都必须保留用户的个人笔记章节
