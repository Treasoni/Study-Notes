# CLAUDE.md - AI 学习知识库

## 项目概述
这是一个用于学习 AI 相关知识的 Obsidian 知识库。

## 学习原则

### 1. 优先使用官方资源
当学习新知识点时，必须按以下顺序获取信息：
1. **官方文档** - 使用 WebSearch 搜索 "xxx official documentation"
2. **官方 GitHub** - 查找最新的 API 变更和示例
3. **权威博客** - 官方博客或核心维护者的文章
4. **社区资源** - 仅作为补充

### 2. 通俗易懂的解释
无论用户要求初级、中级还是高级内容，都必须包含：
- **一句话定义**：简洁的核心概念
- **通俗比喻**：用日常生活中的例子类比
- **具体示例**：可运行的代码或操作步骤
- **图示说明**：使用 ASCII 图或 Mermaid 图表

### 3. 知识关联
- 每个新知识点都要检查与现有笔记的关联
- 自动添加双向 wikilink
- 更新相关笔记的"相关文档"部分

## 自定义命令

### /learn <主题>
学习一个新知识点的工作流程：

1. **搜索官方资源**
   - 使用 WebSearch 搜索 "{主题} official documentation 2026"
   - 使用 WebFetch 获取官方文档核心内容

2. **创建笔记**
   - 在合适的目录下创建笔记文件
   - 使用标准笔记模板（见下方）

3. **建立关联**
   - 搜索知识库中相关内容
   - 添加双向 wikilink
   - 更新 MOC 索引

### /update <文件路径>
更新现有笔记的工作流程：

1. **读取现有内容**
   - 读取指定文件
   - 分析当前结构和内容
   - **识别并标记用户自定义章节**（见下方保护列表）

2. **获取最新信息**
   - 识别笔记主题
   - 使用 WebSearch 搜索官方最新文档
   - 使用 WebFetch 获取更新内容

3. **增量更新（重要）**
   - **仅更新**：技术参数、API 变更、最佳实践、参考资料
   - **禁止修改**：用户自定义章节（见保护列表）
   - 保持原有格式和风格

4. **检查双向链接**
   - 扫描知识库中引用此文件的其他笔记
   - 确保链接正确
   - 添加新的关联链接

#### 🛡️ 用户内容保护列表
以下章节在更新时**必须完整保留**，不得修改或删除：
- `## 个人笔记` / `## My Insights`
- `## 随手记` / `## Quick Notes`
- `## 学习心得` / `## Learnings`
- `## 踩坑记录` / `## Pitfalls`
- `## 待探索` / `## TODO`
- 任何以 `> [!personal]` 开头的 callout 块

### /organize <文件夹路径>
整理文件夹的工作流程：

1. **调用 knowledge-base-organizer skill**
   - 扫描指定文件夹下的所有文件
   - 分析文件类型和内容

2. **孤岛检测（Island Detection）**
   - 扫描整个知识库，识别没有任何入链（Inbound links）的笔记
   - 生成"孤岛笔记"列表
   - 为每个孤岛笔记：
     - 查找语义相关的其他笔记
     - 在相关笔记或 MOC 中添加引用
     - 确保每个笔记至少有一个入链

3. **生成/更新 MOC**
   - 创建或更新文件夹的 MOC 索引
   - 按主题分类整理
   - 添加快速查找表
   - **将孤岛笔记优先展示在 MOC 中**

4. **建立文档关联**
   - 识别相关文档
   - 添加双向 wikilink
   - 更新标签分类

## Obsidian 工作流分工 (Workflow Division)

**强制执行**：**必须**严格遵循以下分工模式：

### 查找与读写 → obsidian-cli
- **用途**：快速定位文件路径、批量操作、属性查询
- **典型场景**：
  - "找到所有包含 `#错题` 标签的笔记"
  - "列出本周的所有每日计划文件"
  - "搜索某个知识点的相关笔记"
- **示例命令**：
  ```bash
  # 搜索包含特定标签的笔记
  obsidian search "tag:错题"

  # 列出每日计划文件
  obsidian list "考研计划/每日计划"
  ```

### 内容生成与重构 → Claude Code
- **用途**：深度分析、逻辑检查、内容重写、跨文件整合
- **典型场景**：
  - "帮我检查 `./考研计划/每日计划/2026-03-27.md` 里的逻辑错误并修正"
  - "重构这节笔记的结构，让知识点更清晰"
  - "把这段内容整理成标准格式"
- **优势**：直接在终端对话，Claude 可读取、分析、修改文件

### 最佳实践搭配 (Best Practice)

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| **管理结构** | obsidian-cli | 创建新笔记、移动分类、修改文件名 — CLI 了解 Obsidian 路径规则 |
| **处理内容** | Claude Code | 编辑改写、润色、语义查找 — Claude 理解内容含义 |

**语义查达示例**（只记得内容，不记得文件名）：
```
用户："我之前写过一篇关于极限的笔记，里面提到过夹逼定理"
Claude：用 Grep 搜索关键词 → 找到匹配文件 → 返回给用户
```

### 协作模式
```
用户需求 → 先用 obsidian-cli 定位文件 → 告诉 Claude 去处理它
```
**示例流程**：
1. 用户："帮我整理今天的错题"
2. Claude：先用 obsidian-cli 找到当日计划文件 → 提取错题 → 调用 `/mistake-book` 整理

### obsidian-cli 不可用时的替代方案

当 `obsidian-cli` 无法连接到 Obsidian 主进程时（常见原因：Obsidian 未运行、版本过旧、CLI 插件未启用），**自动切换**到以下替代方案：

| obsidian-cli 命令 | 替代方案 (Glob + Grep + Read) |
|------------------|------------------------------|
| `obsidian search "tag:错题"` | `Grep pattern="#错题" output_mode=files_with_matches` |
| `obsidian list "考研计划/每日计划"` | `Glob pattern="考研计划/每日计划/*.md"` |
| `obsidian get "path/file.md"` | `Read file_path="path/file.md"` |
| `obsidian get "path.md" --frontmatter` | `Read file_path="path.md"` (读取后解析 YAML) |

**错误检测**：
```bash
# 当返回以下错误时，自动切换到替代方案
# "Unable to connect to main process"
# "Your Obsidian installer is out of date"
```

## 注意事项

1. **年份敏感**：搜索时始终使用当前年份（2026）
2. **链接规范**：使用 `[[文件夹/文件名]]` 格式，不带 .md 后缀
3. **标签规范**：使用小写英文，用连字符分隔
4. **保持简洁**：CLAUDE.md 控制在 150-200 条指令以内
5. **保护用户内容**：任何更新操作都必须保留用户的个人笔记章节
