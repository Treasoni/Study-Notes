# 考研信息收集与文档管理技能

> ⚠️ **免责声明**：本技能收集的信息仅供参考，最终以院校官方发布为准。请在做决策前务必核实官方信息。

## 技能概述

本技能专注于收集中国研究生入学考试（考研）的最新信息，以及创建和管理考研相关的文档。与 `kaoyan-math` 技能不同，本技能关注**信息收集**和**文档管理**，而非具体的数学学科内容生成。

## 核心功能

### 1. 信息收集功能

- **院校招生信息**：招生简章、专业目录、招生人数
- **分数线数据**：历年复试线、国家线、自划线院校分数线
- **专业目录变化**：新增/撤销专业、考试科目调整
- **参考书目信息**：初试/复试参考教材、版本要求
- **复试政策**：复试办法、调剂政策、录取规则

### 2. 文档创建功能

- 创建院校考研指南
- 创建专业课笔记框架
- 创建复习计划表
- 创建资料清单
- 创建信息对比表格

### 3. 文档更新功能

- 更新分数线数据到现有文档
- 更新招生人数和报录比
- 更新参考书目信息
- 添加最新政策通知和变化

## 实施模式（安全机制）

本技能采用分级执行模式，确保用户对文档操作有完全控制权。

| 模式 | 行为 | 文件修改 | 适用场景 |
|------|------|----------|----------|
| **① 仅收集信息** | 收集并展示信息 | ❌ 不创建任何文件 | 用户只想了解信息，不需要保存 |
| **② 生成文档草稿** | 生成内容预览 | ❌ 不保存文件 | 用户想预览文档内容后再决定 |
| **③ 更新现有文档** | 修改已存在文档 | ✅ 需用户确认 | 用户明确要求更新现有文档 |

**工作流程**：
1. 首次执行时，询问用户选择执行模式
2. 根据模式执行相应操作
3. 在模式①或②下，完成后询问是否需要保存/创建文档
4. 在模式③下，显示将要修改的内容并等待确认

## 工作范围（Scope）

本技能支持不同范围的操作，避免一次性处理过多内容。

| 范围 | 行为 | 用例示例 |
|------|------|----------|
| **单院校** | 只处理一个院校/专业 | "收集湖大控制工程信息" |
| **多院校对比** | 收集多个院校并生成对比 | "对比湖大、中南、湖师大的控制工程" |
| **全局更新** | 批量更新所有文档 | "更新所有分数线到2025年数据" |

**范围决策规则**：
- 用户未明确指定时，默认为"单院校"模式
- 涉及3个及以上院校时，自动使用"多院校对比"模式
- "全局更新"需要用户明确指定

## 信息收集策略

### 时间优先级规则

考研信息具有强时效性，必须遵循以下优先级：

```
1. 优先获取最新招生年度信息
2. 若最新未发布 → 标注"暂未发布，使用XX年数据"
3. 若使用历史数据 → 必须在数据旁明确标注年份
```

**文档标注规范**：
```markdown
| 年份 | 复试线 | 备注 |
|------|--------|------|
| 2026 | 暂未发布 | 截至2026-02-25 |
| 2025 | 320分 | |
```

### 来源优先级规则

```
1. 用户明确提供的来源（最高优先级）
2. 官方网站（研招网、院校研究生院官网、学院官网）
3. 其他可靠来源（需标注来源）
```

**来源标注规范**：
- 用户提供的：`来源：用户提供`
- 官方网站：`来源：[官网名称](链接)`
- 其他来源：`来源：[来源名称](链接) ⚠️请核实官方信息`

## 触发条件

### 触发此技能当：

- 用户说"收集考研信息"、"收集XX大学考研信息"
- 用户说"创建考研指南"、"生成考研文档"
- 用户说"更新分数线"、"更新招生信息"
- 用户提到"最新考研政策"、"2025考研"、"2026考研"、"2027考研"
- 用户要求"查找XX大学XX专业"信息
- 用户要求"创建/修改/更新考研文档"
- 用户询问"XX学校XX专业招多少人"、"XX专业分数线"

### 不触发此技能当：

- 普通的知识问答（如"什么是考研"、"考研流程是什么"）
- 数学/英语/政治等科目的具体题目解答（使用 kaoyan-math 或其他技能）
- 没有"信息收集"或"文档创建"意图的一般对话
- 仅要求"帮我做题"、"讲解这道题"

## 工作流程

### 0. 准备阶段（首次执行必选）

#### 确认文档路径配置
```
如果用户已配置考研文档目录 → 使用配置的路径
如果用户未配置 → 询问用户：考研文档应保存到哪个目录？
```

**存储建议**：将用户选择的路径保存为会话配置，后续执行无需重复询问

#### 确认执行模式
```
请选择执行模式：
① 仅收集信息 - 不创建任何文件
② 生成文档草稿 - 生成内容预览，不保存
③ 更新现有文档 - 修改已存在的文档
```

#### 确认工作范围
```
请选择工作范围：
- 单院校（如：湖大控制工程）
- 多院校对比（如：湖大、中南、湖师大）
- 全局更新（如：更新所有分数线）
```

### 1. 信息收集流程

1. **明确需求**：确认目标院校、专业、年份
2. **搜索官方信息**：
   - 优先使用用户提供的来源
   - 其次查找院校研究生院官网
   - 最后查找其他可靠来源
3. **提取关键信息**：招生人数、分数线、参考书目等
4. **验证信息时效性**：
   - 确认信息所属年份
   - 标注数据获取时间
   - 如最新数据未发布，明确标注
5. **记录数据来源**：保存官方链接供后续验证

### 2. 文档创建流程

**仅在执行模式②或③下执行**

1. **选择合适模板**：根据需求选择模板
2. **填充收集的信息**：将收集的信息填入模板
3. **添加数据来源**：标注信息来源链接和数据年份
4. **设置更新时间**：在文档开头标注更新时间
5. **处理文档保存**：
   - 模式②：仅展示预览，不保存
   - 模式③：保存到用户配置的考研文档目录

### 3. 文档更新流程

**仅在执行模式③下执行**

1. **找到现有文档**：在考研目录下定位相关文档
2. **比较新旧信息**：识别哪些数据需要更新
3. **更新数据表格**：修改表格中的数据，保留历史数据
4. **添加更新记录**：在更新记录表中记录变更
5. **标注更新时间**：更新文档头部的更新时间
6. **确认修改**：显示修改内容，等待用户确认

## 注意事项

### 信息准确性
1. **信息时效性**：考研信息每年都会有变化，务必确认信息年份
2. **官方信息优先**：优先采信官方渠道信息，谨慎对待非官方渠道
3. **记录数据来源**：所有信息都应标注来源，方便后续验证
4. **数据年份标注**：使用历史数据时必须标注年份，最新数据未发布时明确说明

### 执行安全
5. **遵循执行模式**：严格按照用户选择的模式执行，不越权操作
6. **确认工作范围**：避免一次性处理过多内容
7. **路径配置**：使用用户配置的路径，不使用硬编码路径

### 内容规范
8. **区分学硕专硕**：同一专业的学硕和专硕在招生、考试、分数线等方面可能不同
9. **关注政策变化**：注意考试科目调整、参考书更新等政策变化
10. **免责声明**：提醒用户以官方信息为准
