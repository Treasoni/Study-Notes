# 整理文件夹

## 执行步骤

### 1. 扫描指定文件夹
使用 Glob 扫描指定文件夹下的所有文件

### 2. 分析文件类型和内容
识别每个文件的类型和主题

### 3. 孤岛检测（Island Detection）
扫描整个知识库，识别没有任何入链（Inbound links）的笔记

### 4. 生成/更新 MOC
创建或更新文件夹的 MOC 索引：
- 按主题分类整理
- 添加快速查找表
- 将孤岛笔记优先展示在 MOC 中

### 5. 建立文档关联
- 识别相关文档
- 添加双向 wikilink
- 更新标签分类

---

## 核心逻辑：先"对齐"，再"融合"

为了不影响质量，不能直接"帮我整理"。需要充当一个高水平的编辑，遵循以下三个步骤：

1. **识别（Identify）**：找出旧笔记中被遗漏的"珍珠"
2. **结构化（Structure）**：确定新旧笔记的主从关系
3. **无损合并（Lossless Merge）**：将遗漏点自然地嵌入新框架

---

## 推荐的提示词模板

### 场景 A：以"新笔记"为主，补充遗漏点

```
我有两份关于 [主题] 的笔记。

笔记 A（新笔记）：是我提取出的精简版，目前的结构更清晰。
笔记 B（旧笔记）：虽然篇幅长且乱，但里面包含了一些我在 A 中遗漏的深度思考和细节。

任务：请对比这两份笔记。在保持笔记 A 现有逻辑结构的前提下，找出笔记 B 中那些"没被 A 包含但有价值"的信息，并将它们自然地融合进 A。

要求：
1. 不要删除 A 中的任何现有逻辑
2. 如果 B 中的信息与 A 重合，请以 A 的表述为主
3. 请在合并处加粗或用"[补充]"标注，方便我核对
4. 保持我一贯的记录语气
```

### 场景 B：不确定如何取舍，需要大纲对比

```
请帮我分析这两份有重合部分的笔记。
请先列出一个对比表格，说明：
- 哪些信息是 A 独有的
- 哪些信息是 B 独有的
- 哪些是重合的

最后，请根据这两份笔记，为我建议一个更科学、更完整的新大纲，确保没有任何旧笔记中的有用细节被丢弃。
```

---

## 避免"笔记质量下降"的小技巧

在使用 AI 整理时，最怕它把个人感悟变成了 AI 味十足的废话。建议：

1. **保留"毛刺"**：告诉 AI "保留我的口语化表达和非正式的感悟"，这些才是笔记的精华

2. **分段处理**：如果笔记总字数超过 5000 字，建议分章节发送，否则 AI 容易出现"幻觉"或漏掉结尾的细节

3. **明确"有用"的标准**：比如你可以说"我特别在意旧笔记里的案例和数据，请务必保留"

---

## 进阶建议：建立"引用"而非"搬运"

如果因为篇幅太长才分开写，说明需要的是**双向链接（Backlinks）**的思维。

可以要求：

```
请在整理新笔记时，在相关段落末尾帮我写下：
"参考旧笔记中关于 [具体细节] 的部分"
这样我回顾时就能定位回去。
```
