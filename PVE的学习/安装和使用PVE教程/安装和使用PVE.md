# PVE 安装和配置完整教程

> [!info] 概述
> Proxmox Virtual Environment (PVE) 是一个基于 Debian 的开源虚拟化管理平台，支持 KVM 虚拟机和 LXC 容器。
> 类比：就像一个"装了所有虚拟化软件的 Debian 系统"，开箱即用，自带 Web 管理界面。

## 核心概念 💡
- **PVE**：基于 Debian 的虚拟化管理平台
- **BIOS 虚拟化设置**：CPU 虚拟化和设备直通的基础
- **网络配置**：安装时配置 IP 地址、网关等关键信息
- **WinPE**：用于清理硬盘、统一分区表格式的辅助工具

## 操作步骤

### 1. 下载 PVE 系统及写盘软件

#### 1.1 下载 PVE 镜像

官方下载地址：https://www.proxmox.com/en/downloads

**当前最新版本**：Proxmox VE 9.1-1（发布于 2025 年 11 月 19 日，文件大小约 1.83 GB）

SHA256SUM: `6d8f5afc78c0c66812d7272cde7c8b98be7eb54401ceb045400db05eb5ae6d22`

选择最新版本的 ISO 镜像下载。

#### 1.2 准备写盘工具

根据需求选择写盘工具：
- **Rufus**：成功率最高，支持绕过 Win11 限制
- **balenaEtcher**：最简单，适合新手
- **Ventoy**：多系统启动，灵活方便

详细对比：[[写盘工具]]

### 2. 准备 WinPE（可选但推荐）

#### 为什么需要 WinPE？

> [!warning] 重要说明
> PVE 官方安装器**技术上不需要** WinPE，但**现实中经常需要**。

PVE 安装器对磁盘的"历史遗留问题"很敏感，可能出现：

| 问题现象 | 原因 |
|---------|------|
| 安装界面看不到硬盘 | 有旧的分区表/引导信息 |
| 选择硬盘后报错 | 分区表格式混乱（GPT/MBR 混合） |
| ZFS/LVM 创建失败 | 有隐藏分区或残留数据 |
| 装完无法启动 | 引导信息冲突 |

#### WinPE 的作用

1. **清空旧分区表**（最重要）
2. **统一分区表格式**（GPT/MBR）
3. **删除隐藏/保留分区**（EFI、MSR、恢复分区）

#### 推荐 WinPE 工具

| 工具 | 下载地址 | 当前版本 |
|------|----------|----------|
| **FirPE** | https://www.firpe.cn | V2.0.3（2025.04.03） |
| 微PE | https://www.wepe.com.cn | - |

> [!tip] 注意
> 下载 **ISO 格式**的 WinPE，不要下载 exe 版本

### 3. 进入 BIOS 系统

#### 3.1 进入 BIOS 方法

开机时连续按以下按键进入 BIOS：
- `Delete` 键
- `F10` 键
- `F2` 键
- `F12` 键

具体按键取决于主板品牌。

#### 3.2 设置 U 盘启动优先级

1. 在 BIOS 中找到 `Boot` 或 `启动` 菜单
2. 将 U 盘设置为第一启动项
3. **禁用 Secure Boot**（推荐）

### 4. BIOS 虚拟化设置（重要）

#### 4.1 VT / SVM（最重要）

| 品牌 | 选项名称 |
|------|----------|
| Intel | **VT-x** |
| AMD | **SVM** / **AMD-V** |

**作用**：让 CPU 能跑虚拟机

**不开会怎样**：
- ❌ PVE/ESXi/VMware 虚拟机**起不来**
- ❌ Docker 里跑 KVM 不行
- ❌ Windows/Linux 虚拟机性能极差（软件模拟）

📌 **结论**：👉 装虚拟机 = 必开

#### 4.2 VT-d / AMD-Vi / IOMMU（进阶神器）

| 品牌 | 选项名称 |
|------|----------|
| Intel | **VT-d** |
| AMD | **IOMMU** / **AMD-Vi** |

**作用**：设备直通（PCI Passthrough）

也就是把物理硬件直接给虚拟机用：
- 🖥 显卡直通 → 虚拟 Windows 打游戏
- 💾 SATA 控制器直通 → TrueNAS
- 🌐 网卡直通 → 软路由

📌 **结论**：👉 想玩直通 = 必开，不用现在玩也建议**提前开**

#### 4.3 Above 4G Decoding

**作用**：允许 PCIe 设备使用 4GB 以上地址空间

**适用**：PCIe / 高端设备支持（直通增强）

**不开会怎样**：
- ❌ 显卡直通容易失败
- ❌ SR-IOV 不工作
- ❌ 新显卡识别异常

📌 **结论**：👉 只要玩直通/显卡 = 必开

#### 4.4 SR-IOV（高端玩法）

**作用**：一块物理网卡 → 变成多块"虚拟网卡"

**适合**：
- 企业级网卡（Intel X520/X710）
- PVE/ESXi
- 高性能软路由

📌 **结论**：👉 有支持的网卡才有用，家用板子有就开

### 5. 安装配置 PVE

#### 5.1 从 WinPE 开始（推荐）

1. 启动进入 Ventoy 菜单
2. 先选择 WinPE 进入
3. 使用磁盘工具（DiskGenius）：
   - 删除所有分区
   - 转换分区表类型（GPT 用于 UEFI，MBR 用于 Legacy）
   - 保存更改
4. 重启

#### 5.2 开始 PVE 安装

1. 从 PVE 启动盘启动
2. 选择 `Install Proxmox VE`
3. 同意许可协议
4. 选择目标硬盘（已清理）
5. 配置网络信息（重要）

#### 5.3 网络配置（关键）

安装时的网络配置：

```
IP 地址：192.168.x.x
子网掩码：255.255.255.0
网关：192.168.x.1
DNS：192.168.x.1
```

> [!warning] 重要提示
> 配置错误或把主机放到别的网段后，看 [[修改PVE的网络信息]] 教程进行修改

### 6. 完成安装

1. 等待安装完成
2. 重启系统
3. 使用浏览器访问：`https://你的IP:8006`
4. 首次登录可能提示证书警告，点击"继续访问"

## BIOS 设置总结

你配置的这套选项意味着：

- ✅ **CPU 已经完全支持虚拟化**
- ✅ **PVE/ESXi 能满血跑**
- ✅ **显卡/网卡/硬盘都能直通**
- ✅ **NAS/软路由/Windows VM 都能玩**
- ✅ **这是服务器级配置思路**

> **一句话总结："我这不是普通家用机，这是准服务器"**

## 注意事项 ⚠️

1. **安装前备份数据**：安装 PVE 会清空目标硬盘
2. **网络配置要正确**：错误的 IP 会导致无法访问 Web 界面
3. **BIOS 设置要保存**：修改后记得保存（F10）并重启
4. **Secure Boot 建议关闭**：避免兼容性问题
5. **虚拟化选项必须开启**：VT-x/VT-d 是 PVE 的基础

## 常见问题 ❓

**Q: 安装界面看不到硬盘？**
A: 硬盘可能有旧的分区表，先用 WinPE 清理。

**Q: 安装后无法访问 Web 界面？**
A: 检查网络配置，确认 IP 地址正确，用 [[修改PVE的网络信息]] 修改。

**Q: BIOS 里找不到 VT-d 选项？**
A: 不是所有主板都支持，查阅主板手册确认。

**Q: 安装时提示"无法安装引导加载程序"？**
A: 确认硬盘分区表格式与 BIOS 模式匹配（UEFI+GPT，Legacy+MBR）。

**Q: PVE 和 ESXi 哪个好？**
A: PVE 开源免费、功能丰富、社区活跃；ESXi 更稳定但有商业限制。家用推荐 PVE。
