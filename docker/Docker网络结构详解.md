## 一、Docker 网络架构概览

### 1.1 是什么

**Docker 网络** 是 Docker 提供的一套网络解决方案，让容器��够：
- 容器之间相互通信
- 容器与宿主机通信
- 容器访问外部网络
- 外部网络访问容器

### 1.2 通俗理解

**🎯 比喻**：Docker 网络就像「公司的通信系统」

## 二、底层实现原理

### 2.1 核心技术栈

| 技术 | 作用 | 比喻 |
|------|------|------|
| **Network Namespace** | 网络隔离，每个容器独立的网络栈 | 每个员工独立的办公桌 |
| **veth pair** | 虚拟网卡对，连接容器和网桥 | 电话线（两端各一个接口）|
| **Bridge** | 虚拟网桥，连接多个 veth | 电话交换机 |
| **iptables** | NAT 地址转换、端口映射 | 总机转接系统 |

### 2.2 veth pair 原理

**🎯 比喻**：veth pair 就像「一根虚拟网线」

```
┌─────────────────────────────────────────────────────────────┐
│                      veth pair 工作原理                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│   │              │                    │              │      │
│   │  ┌────────┐  │    veth pair      │  ┌────────┐  │      │
│   │  │ eth0   │◄─┼────────────────────┼─►│vethxxx │  │      │
```

**🎯 比喻**：NAT 就像「公司总机」

```
┌─────────────────────────────────────────────────────────────┐
│                    NAT 地址转换原理                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  场景 1：容器访问外网 (SNAT - 源地址转换)                     │
│  ─────────────────────────────────────────                  │
│                                                             │
│  容器 A (172.17.0.2)                                        │
│       │                                                     │
│       │ 请求: src=172.17.0.2, dst=8.8.8.8                   │
│       ▼                                                     │
│  ┌─────────┐                                                │
│  │ docker0 │                                                │
│  │ (网关)  │                                                │
│  └────┬────┘                                                │
│       │                                                     │
│       │ iptables SNAT                                       │
│       │ 请求: src=宿主机IP, dst=8.8.8.8  ← 源地址被替换      │
│       ▼                                                     │
│  ┌─────────┐                                                │
│  │  eth0   │ ────► Internet (8.8.8.8)                       │
│  │ (物理)  │                                                │
│  └─────────┘                                                │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  场景 2：外部访问容器 (DNAT - 目标地址转换)                   │
│  ─────────────────────────────────────                      │
│                                                             │
│  Internet                                                   │
│       │                                                     │
│       │ 请求: dst=宿主机IP:8080                              │
│       ▼                                                     │
│  ┌─────────┐                                                │
│  │  eth0   │                                                │
│  │ (物理)  │                                                │
│  └────┬────┘                                                │
│       │                                                     │
│       │ iptables DNAT                                       │
│       │ 请求: dst=172.17.0.2:80  ← 目标地址被替换            │
│       ▼                                                     │
│  ┌─────────┐                                                │
│  │ docker0 │                                                │
│  └────┬────┘                                                │
│       │                                                     │
│       ▼                                                     │
│  容器 A (172.17.0.2:80)                                     │
│                                                             │

```


## 三、网络模式详解

### 3.1 网络模式总览

| 模式 | 命令参数 | 特点 | 适用场景 |
|------|---------|------|---------|
| **bridge** | `--network bridge` | 默认模式，独立网络命名空间 | 大多数应用 |
| **host** | `--network host` | 共享宿主机网络 | 高性能、网络监控 |
| **none** | `--network none` | 无网络 | 安全隔离 |
| **overlay** | `--network overlay` | 跨主机通信 | Swarm 集群 |
| **macvlan** | `--driver macvlan` | 容器有独立 MAC 地址 | 需要物理网络直接访问 |

### 3.2 Bridge 模式（默认）

**🎯 比喻**：就像「公司内网」

```
┌─────────────────────────────────────────────────────────────┐
│                    Bridge 模式                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特点：                                                      │
│  ✅ 每个容器有独立 IP                                        │
│  ✅ 容器间可通过 IP 互访                                      │
│  ✅ 需要端口映射才能被外部访问                                │
│  ✅ 默认网络：172.17.0.0/16                                  │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │                    docker0 网桥                        │ │
│  │                   172.17.0.1                          │ │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐               │ │
│  │  │ 容器 A  │  │ 容器 B  │  │ 容器 C  │               │ │
│  │  │172.17.0.2│  │172.17.0.3│  │172.17.0.4│               │ │
│  │  │  :80    │  │  :3306  │  │  :6379  │               │ │
│  │  └─────────┘  └─────────┘  └─────────┘               │ │
│  └───────────────────────────────────────────────────────┘ │
│                         │                                   │
│                    端口映射                                  │
│                    8080→80                                  │
│                         │                                   │
│                      外部访问                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**使用示例**：

```bash
# 创建容器（默认 bridge 模式）
docker run -d --name web nginx

# 端口映射：宿主机 8080 → 容器 80
docker run -d --name web -p 8080:80 nginx

# 访问：http://宿主机IP:8080
```

**容器间通信**：

```bash
# 在同一 bridge 网络中，容器可通过 IP 互访
# 但不能通过容器名访问（除非使用自定义网络或 --link）

# 创建自定义 bridge 网络（推荐）
docker network create mynet

# 使用自定义网络的容器可以通过容器名互访
docker run -d --name app --network mynet myapp
docker run -d --name db --network mynet mysql

# app 容器中可以直接访问 db:3306
```

> - [深入理解Docker网络：Bridge、Host、Overlay与None模式详解](https://www.cnblogs.com/jzssuanfa/p/19285156) - 博客园

### 3.3 Host 模式

**🎯 比喻**：就像「共用办公室」

```
┌─────────────────────────────────────────────────────────────┐
│                    Host 模式                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特点：                                                      │
│  ✅ 容器共享宿主机网络命名空间                                │
│  ✅ 无网络隔离，性能最高                                      │
│  ✅ 容器直接使用宿主机端口                                    │
│  ❌ 端口冲突风险（容器端口不能重复）                           │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │                     宿主机网络                         │ │
│  │                    192.168.1.100                      │ │
│  │                                                       │ │
│  │   ┌─────────────────────────────────────────────┐    │ │
│  │   │         容器 A (Host 模式)                   │    │ │
│  │   │         直接使用宿主机端口                   │    │ │
│  │   │         :80, :443, :8080                    │    │ │
│  │   └─────────────────────────────────────────────┘    │ │
│  │                                                       │ │
│  │   ┌─────────────────────────────────────────────┐    │ │
│  │   │         容器 B (Host 模式)                   │    │ │
│  │   │         不能使用 :80（已被占用）             │    │ │
│  │   │         可以使用 :3000                      │    │ │
│  │   └─────────────────────────────────────────────┘    │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
│  访问：直接 http://192.168.1.100:80（无需端口映射）           │
│                                                             │
└─��───────────────────────────────────────────────────────────┘
```

**使用示例**：

```bash
# Host 模式运行
docker run -d --name web --network host nginx

# 访问：http://宿主机IP:80（无需 -p 映射）
```

**适用场景**：
- 需要高性能网络（无 NAT 开销）
- 网络监控工具
- 需要访问宿主机所有端口

> [!warning] 注意
> Host 模式下容器没有网络隔离，安全风险较高

```bash
# 验证：只有 loopback
docker exec isolated ip addr
# 1: lo: <LOOPBACK,UP,LOWER_UP> ...
```

**🎯 比喻**：就像「跨公司专线」

```
┌─────────────────────────────────────────────────────────────┐
│                    Overlay 模式                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特点：                                                      │
│  ✅ 跨主机容器通信                                           │
│  ✅ 基于 VXLAN 隧道封装                                      │
│  ✅ 用于 Docker Swarm 集群                                   │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │                    Overlay 网络                         ││
│  │                   (跨主机虚拟网络)                       ││
│  │                                                        ││
│  │    ┌─────────────────┐      ┌─────────────────┐       ││
│  │    │    主机 A       │      │    主机 B       │       ││

│  │    │  ┌─────────┐   │      │  ┌─────────┐   │       ││
│  │    │  │ 容器 A  │   │◄────►│  │ 容器 B  │   │       ││
│  │    │  │10.10.0.2│   │ VXLAN │  │10.10.0.3│   │       ││

│  └─────────────────────────────────────────────────────────┘│

```

**使用示例**（需要 Swarm）：

```bash
# 初始化 Swarm
docker swarm init

# 创建 overlay 网络
docker network create --driver overlay my-overlay

# 在 overlay 网络中创建服务
docker service create --name web --network my-overlay nginx
```

## 四、Docker 网络命令

### 4.1 网络管理命令速查表

| 命令 | 作用 | 示例 |
|------|------|------|
| `docker network ls` | 列出所有网络 | `docker network ls` |
| `docker network create` | 创建网络 | `docker network create mynet` |
| `docker network rm` | 删除网络 | `docker network rm mynet` |
| `docker network inspect` | 查看网络详情 | `docker network inspect bridge` |
| `docker network connect` | 连接容器到网络 | `docker network connect mynet container1` |
| `docker network disconnect` | 断开容器网络 | `docker network disconnect mynet container1` |
| `docker network prune` | 删除未使用网络 | `docker network prune` |

### 4.2 常用命令详解

#### 列出网络

```bash
docker network ls

# 输出：
# NETWORK ID     NAME      DRIVER    SCOPE
# abc123def456   bridge    bridge    local
# def456abc123   host      host      local
# 123abc456def   none      null      local
```

#### 创建自定义网络

```bash
# 创建 bridge 网络（最常用）
docker network create mynet

# 指定子网
docker network create --subnet=192.168.100.0/24 mynet

# 指定网关
docker network create --subnet=192.168.100.0/24 --gateway=192.168.100.1 mynet

# 限制 IP 范围
docker network create --subnet=192.168.100.0/24 --ip-range=192.168.100.128/25 mynet
```

#### 查看网络详情

```bash
docker network inspect bridge

# 输出包含：
# - 网络配置（子网、网关）
# - 连接的容器列表
# - IP 地址分配情况
```

#### 容器连接网络

```bash
# 创建时指定网络
docker run -d --name app --network mynet nginx

# 运行后连接到网络
docker network connect mynet existing-container

# 指定固定 IP
docker run -d --name app --network mynet --ip 192.168.100.10 nginx

# 断开网络
docker network disconnect mynet app
```

### 4.3 端口映射

```bash
# 映射单个端口
docker run -d -p 8080:80 nginx

# 映射多个端口
docker run -d -p 8080:80 -p 8443:443 nginx

# 指定 IP 映射
docker run -d -p 127.0.0.1:8080:80 nginx

# 映射 UDP 端口
docker run -d -p 53:53/udp dns-server

# 随机端口映射
docker run -d -P nginx
# 查看映射端口：docker port container_name
```

## 五、实战场景与最佳实践

### 5.1 场景一：Web 应用 + 数据库

**需求**：Web 应用需要连接数据库

**方案**：使用自定义 bridge 网络

```yaml
# docker-compose.yml
version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "8080:80"
    networks:
      - appnet
    depends_on:
      - db

  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: secret
    networks:
      - appnet
    volumes:
      - dbdata:/var/lib/mysql

networks:
  appnet:
    driver: bridge

volumes:
  dbdata:
```

**优点**：
- Web 可以通过服务名 `db` 访问数据库
- 网络隔离，外部无法直接访问数据库
- 易于扩展

### 5.2 场景二：微服务架构

**需求**：多个微服务需要相互通信

**方案**：按业务域划分网络

```yaml
version: '3.8'

services:
  # 前端服务
  frontend:
    image: frontend
    networks:
      - frontend-net

  # API 网关
  api-gateway:
    image: api-gateway
    networks:
      - frontend-net
      - backend-net

  # 用户服务
  user-service:
    image: user-service
    networks:
      - backend-net
      - db-net

  # 订单服务
  order-service:
    image: order-service
    networks:
      - backend-net
      - db-net

  # 数据库
  db:
    image: postgres
    networks:
      - db-net

networks:
  frontend-net:
  backend-net:
  db-net:
```

**网络拓扑**：

```
┌─────────────────────────────────────────────────────────────┐
│                    微服务网络架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐                                           │
│  │ frontend-net│                                           │
│  │  frontend   │◄──────────┐                               │
│  └─────────────┘           │                               │
│                            │                               │
│                    ┌───────▼───────┐                       │
│                    │  api-gateway  │                       │
│                    └───────┬───────┘                       │
│                            │                               │
│  ┌─────────────┐    ┌──────▼──────┐    ┌─────────────┐    │
│  │ backend-net │◄───┤             ├───►│   db-net    │    │
│  │user-service│    │             │    │     db      │    │
│  │order-service│   │             │    │             │    │
│  └─────────────┘    └─────────────┘    └─────────────┘    │
│                                                             │
│  优点：                                                      │
│  - 前端不能直接访问数据库（安全）                              │
│  - 服务间通信通过网关                                        │
│  - 数据库网络完全隔离                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 最佳实践

#### ✅ 推荐做法

| 实践 | 说明 |
|------|------|
| **使用自定义网络** | 避免使用默认 bridge，自定义网络支持 DNS 解析 |
| **按业务划分网络** | 不同安全级别的服务放在不同网络 |
| **使用 Docker Compose** | 自动创建和管理网络 |
| **固定重要服务 IP** | 数据库等核心服务使用固定 IP |
| **端口映射最小化** | 只暴露必要端口 |

#### ❌ 避免的做法

| 避免 | 原因 |
|------|------|
| 使用 `--link` | 已废弃，用自定义网络替代 |
| 所有容器同一网络 | 安全风险 |
| 过度使用 host 模式 | 无网络隔离 |
| 端口映射过多 | 增加攻击面 |

## 六、常见问题排查

### 6.1 问题速查表

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 容器间无法通信 | 不同网络 | 连接到同一网络 |
| 无法通过容器名访问 | 默认 bridge | 使用自定义网络 |
| 外部无法访问容器 | 未映射端口 | 添加 `-p` 参数 |
| 端口冲突 | 端口被占用 | 更换端口或停止占用进程 |
| DNS 解析失败 | DNS 配置问题 | 检查 `--dns` 配置 |

### 6.2 排查命令

```bash
# 1. 检查容器网络配置
docker inspect container_name | grep -A 20 "NetworkSettings"

# 2. 进入容器测试网络
docker exec -it container_name sh
# 容器内执行：
ping other_container
curl http://service:port
nslookup service_name

# 3. 查看网络详情
docker network inspect network_name

# 4. 检查端口监听
netstat -tlnp | grep :8080

# 5. 查看 iptables 规则
sudo iptables -t nat -L -n

# 6. 查看容器 IP
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container_name
```

### 6.3 常见错误解决

#### 错误 1：容器间 ping 不通

```bash
# 原因：不在同一网络
# 解决：连接到同一网络

docker network connect mynet container1
docker network connect mynet container2
```

#### 错误 2：无法通过容器名访问

```bash
# 原因：使用默认 bridge 网络
# 解决：使用自定义网络

docker network create mynet
docker run --network mynet --name app1 ...
docker run --network mynet --name app2 ...

# 现在 app1 可以通过 "app2" 访问 app2
```

#### 错误 3：端口已被占用

```bash
# 错误信息：
# Error: bind: address already in use

# 查找占用进程
lsof -i :8080

# 解决方案 1：停止占用进程
kill -9 <PID>

# 解决方案 2：更换端口
docker run -p 8081:80 nginx
```
