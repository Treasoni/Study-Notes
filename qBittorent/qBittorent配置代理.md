## 1.2 代理服务器设置这样填👇
**混合代理（重点）**

`192.168.110.119:7893`
这是：

- Clash 的 **代理入口**
- 同时支持：
    - HTTP
    - SOCKS5

![](assets/qBittorent配置代理/截屏2026-02-02%2022.35.34.png)

| 项目  | 值                                    |
| --- | ------------------------------------ |
| 类型  | **HTTP**                             |
| 主机  | **iStoreOS 的 IP**（如 192.168.110.119） |
| 端口  | **7893**                             |
| 用户名 | 设了鉴权就添                               |
| 密码  | 设了鉴权就添                               |

> [!warning]
> ⚠️ 这是重点警告
> qb → iStoreOS 上的 Clash（局域网），这里是否需要密码看[2.3 什么时候【需要】账号密码？](../docker/docker进行代理.md#2.3%20什么时候【需要】账号密码？)

## 1.3 测试代理是否生效

**方法 1：查看 Tracker 状态**

1. 添加一个需要代理的 Tracker（如公共 Tracker）
2. 等待几秒，查看状态是否变为「Working」

**方法 2：使用测试种子**

- 添加一个国外公开站点的种子
- 观察是否能成功连接 Tracker

**方法 3：检查日志**

**视图 → 日志**

查看是否有关于代理连接的信息

# 2. 代理常见问题

## 2.1 代理连接失败

**症状：**
- Tracker 状态一直是「Not Working」
- 无法访问国外站点

**排查：**

| 步骤 | 操作 |
|------|------|
| 1 | 检查 iStoreOS 是否正常运行 |
| 2 | 检查 Clash 是否启动 |
| 3 | 测试端口 7893 是否可访问 `telnet 192.168.110.119 7893` |
| 4 | 检查 Clash 日志是否有连接请求 |
| 5 | 确认 Clash 允许局域网连接 |

## 2.2 添加认证（如果需要）

如果你的代理需要认证：

| 项目  | 值  |
| --- | --- |
| 用户名 | 填入你的代理用户名 |
| 密码  | 填入你的代理密码 |

## 2.3 需要代理的流量

qBittorrent 代理主要影响：

- ✅ Tracker 连接
- ✅ DHT 查询（部分情况）
- ✅ WebUI 访问（如果外部访问）

> [!note]
> **注意：** P2P 数据下载通常不经过代理（这是正常的）
> 代理主要用于连接 Tracker 服务器，获取 Peers节点

# 3. 仅对 Tracker 使用代理

如果你想**只让 Tracker 走代理**，而 P2P 直连：

**设置 → 连接**

- 代理服务器：填写你的代理
- 勾选「仅对 Tracker 使用代理」

这种方式的优缺点：

| 方式 | 优点 | 缺点 |
|------|------|------|
| 全局代理 | Tracker 和 DHT 都能访问 | 可能影响下载速度 |
| 仅 Tracker | 下载不受代理影响 | DHT 可能无法正常工作 |

# 4. Clash 配置参考

如果你需要配置 Clash 允许局域网代理，编辑 `config.yaml`：

```yaml
mixed-port: 7893
allow-lan: true
bind-address: "*"
```

重启 Clash 后生效。
